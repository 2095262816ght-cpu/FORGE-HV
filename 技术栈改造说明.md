# FORGE-HV 技术栈改造说明

> 文档说明：本文记录本项目从「单体 Python 实验台」改造为「Python + Vue + Java Spring Boot + MySQL」四栈融合架构的变更内容。  
> 适用场景：答辩展示、交接说明、二次开发入门。

---

## 1. 改造前后对比

| 维度 | 改造前 | 改造后 |
|------|--------|--------|
| 前端 | 静态 HTML + CSS + JS（`frontend/`） | **Vue 3** + Vite + Element Plus + ECharts（`frontend-vue/`） |
| 业务后端 | Flask 一体（`app.py` 同时管页面 API、鉴权、ML） | **Spring Boot 3**（JWT 鉴权、用户 / 历史 / 设置） |
| 数据库 | SQLite（`forge.db`） | **MySQL 8**（库名 `forge_hv`） |
| 机器学习 | Flask 内嵌算法逻辑 | **Python Flask 微服务**（`ml-service/`，复用原算法代码） |
| 启动方式 | `python app.py` 单进程 | 三服务并行：Vue :5173 · Spring :8080 · ML :5001 |
| UI 风格 | Apple / iOS 浅色主题 | **恢复**原 Python 深色主题（iOS Dark Mode） |

**改造原则：** 不削弱原有 DDPG / GAN / sklearn 算法能力，只把「界面、鉴权、持久化」拆到更合适的技术层。

## 2. 目标架构

```
浏览器 Vue 3 (:5173)
   │  Axios · JWT · /api/*
   ▼
Spring Boot 3 (:8080)
   ├── JPA / Hibernate ──► MySQL (:3306)  forge_hv
   │                         · users
   │                         · history
   │                         · settings
   └── RestClient 代理 ──► Python ML (:5001)
                              · 复用 app.py / DDPG.py / GAN-main.py / CT_main.py …
                              · 训练 · 对比 · 异常值 · 相关性 · 数据读写
```

| 层 | 技术选型 | 目录 | 端口 | 职责 |
|----|----------|------|------|------|
| 前端 | Vue 3.5 · Vite 5 · Vue Router · Pinia · Element Plus · ECharts · Axios | `frontend-vue/` | 5173 | 实验台 UI、登录注册、图表交互 |
| 业务后端 | Spring Boot 3.2 · Spring Security · JWT (jjwt) · Spring Data JPA | `backend/` | 8080 | 鉴权、用户管理、训练历史、系统设置；其余 ML 接口透传 |
| 数据库 | MySQL 8 · utf8mb4 | `sql/init.sql` | 3306 | 业务数据持久化 |
| ML 引擎 | Python 3.10+ · Flask · PyTorch · scikit-learn · pandas | `ml-service/` + 根目录算法 | 5001 | 全部机器学习与数据处理 |

## 3. 各层技术细节

### 3.1 前端（Vue 3）

| 项目 | 说明 |
|------|------|
| 框架 | Vue 3（Composition API + `<script setup>`） |
| 构建 | Vite |
| 路由 | Vue Router 4（页面级路由，对应原 SPA 的 `data-page`） |
| 状态 | Pinia（登录态 / Token） |
| UI 组件 | Element Plus（表格、表单、对话框等） |
| 图表 | ECharts（热力/气泡矩阵、训练曲线、散点对比等） |
| HTTP | Axios，统一走 `/api`，自动带 JWT |
| 视觉 | 复用原 `frontend/styles.css` 的 Apple / iOS 浅色主题 |

主要页面（与论文章节对齐）：

- 数据可视化、异常值检测、元素相关性、数据库管理  
- DDPG 训练、DDPG 模型架构、GAN 数据生成过程  
- 5.3 硬度预测对比、5.4 GAN 数据扩充对比  
- 数据管理、历史记录、用户管理、系统设置  

### 3.2 业务后端（Spring Boot）

| 项目 | 说明 |
|------|------|
| 版本 | Spring Boot **3.2.5** · Java **17** |
| 安全 | Spring Security + JWT Filter |
| 持久化 | Spring Data JPA + MySQL Connector/J |
| 代理 | RestClient 转发到 `http://127.0.0.1:5001` |
| 包结构 | `com.forge.hv`（controller / service / entity / repository / security / config） |

Spring 自管（不转发 ML）的能力：

- 登录 / 注册 / 游客 / 改密  
- 用户 CRUD（管理员）  
- 训练历史读写（训练成功后由代理层写入 MySQL）  
- 系统设置（如是否允许游客、是否开放注册）  

其余 `/api/**`（训练、对比、相关性、数据导入导出等）由 `MlProxyController` 代理到 Python。

### 3.3 数据库（MySQL）

库名：`forge_hv`

| 表 | 用途 |
|----|------|
| `users` | 账号、角色（admin / user / guest）、显示名、邮箱 |
| `history` | 训练 / 实验历史（算法、指标 JSON、耗时、状态） |
| `settings` | 键值对系统配置 |

初始化：

```bash
mysql -u root -p < sql/init.sql
```

本机密码可通过 `application-local.yml` 或环境变量覆盖：

- `FORGE_DB_URL` / `FORGE_DB_USER` / `FORGE_DB_PASSWORD`

### 3.4 ML 微服务（Python）

| 项目 | 说明 |
|------|------|
| 框架 | Flask（轻量 API，无页面） |
| 算法 | DDPG、GAN、LinearRegression、PolynomialRegression、SVR |
| 指标 | RMSE / MAE / R² / MAPE |
| 数据 | 149 条实测 + GAN 扩充样本；支持 Excel/CSV 导入 |
| 依赖 | 见 `requirements.txt` / `ml-service/requirements.txt` |

**说明：** 根目录 `app.py`、`frontend/` 仍保留作备份与算法源码复用；日常主入口改为四栈启动脚本。

## 5. 端口与启动

| 服务 | 地址 | 启动 |
|------|------|------|
| Vue | http://localhost:5173 | `scripts/start-frontend.bat` 或 `cd frontend-vue && npm run dev` |
| Spring Boot | http://localhost:8080 | `scripts/start-backend.bat` 或 `cd backend && mvn spring-boot:run` |
| Python ML | http://127.0.0.1:5001 | `scripts/start-ml.bat` 或 `python ml-service/main.py` |
| MySQL | localhost:3306 | 系统服务 |

一键（分别开三个窗口）：

```bat
scripts\start-all.bat
```

默认管理员：`admin` / `admin123`

健康检查：

- `GET http://localhost:8080/api/health`

## 6. 改造收益

1. **职责分离**：UI、业务鉴权、算法训练互不拖累，便于单独扩缩与排错。  
2. **数据可靠**：用户与历史进 MySQL，避免 SQLite 单文件并发与迁移不便。  
3. **安全边界清晰**：JWT 由 Spring Security 统一校验；ML 服务内网调用。  
4. **前端可维护**：组件化、路由化，图表与页面结构更易迭代。  
5. **算法资产保留**：原 Python 科学计算栈完整保留，论文实验流程不断档。  
6. **视觉连续**：Vue 界面复刻原 Python 前端风格，降低使用与演示成本。

## 9. 安全加固备忘（v3.2）

- 游客 JWT 仅可 GET；训练 / 上传 / 写数据需 USER/ADMIN  
- `/api/data/**` 不再匿名开放；需登录  
- 密码改为 BCrypt（旧 SHA-256 登录成功后自动升级）  
- JWT secret 支持 `FORGE_JWT_SECRET`；CORS 限定本机前端源  
- 历史记录按用户隔离（管理员可见全部）  
- 对比实验散点绑定本次 API；论文预置数据明确标注
